# Contributing to PaperLens

## Branching

- `main`: stable
- `dev`: integration
- `feature/<short-name>`: feature work
- `fix/<short-name>`: bug fixes
- `experiment/<short-name>`: research experiments

## Definition of Done

A change is done when:

- implementation is complete
- tests pass locally
- CI passes
- docs are updated
- `docs/PROJECT_CONTEXT.md` is updated
- limitations are documented when relevant
- evaluation impact is known or explicitly scheduled

## Commit style

Use concise conventional-style commits:

```text
feat(api): add document upload route
fix(retrieval): preserve page number metadata
docs(thesis): define research questions
exp(table): compare markdown vs structured table retrieval
```
